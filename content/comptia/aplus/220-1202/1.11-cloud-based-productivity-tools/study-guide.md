# Objective 1.11: Cloud-Based Productivity Tools

Objective 1.11 asks you to install and configure cloud productivity services for
a stated user or business need. The key task is to match the need to the right
service, then configure storage, identity, and licensing so the user receives
the intended access. (CompTIA Objective 1.11; Professor Messer, p. 23)

## Cloud Email Systems

A cloud email system hosts the mail service outside the organization's local
email-server infrastructure. The provider-operated service can combine mailbox
access with connectivity, redundancy, backup, and centralized security
functions. For the technician, the durable concept is the service location and
the user's access to the hosted mailbox; provider-specific setup screens and
product names can change. (Professor Messer, p. 23)

## Cloud Storage and Folder Settings

Cloud storage can keep a user's files available across registered devices.
Synchronization copies a locally saved change to cloud storage and then makes
the updated file available to the user's other synchronized devices.

Folder settings determine what is synchronized and how it is accessed. A user
may keep selected folders stored locally for offline access while leaving other
content available through on-demand or streaming access. Selective
synchronization is especially useful when a device has less local storage than
the user's cloud library. (Professor Messer, p. 23)

## Collaboration Tools

Cloud collaboration tools let people work together in real time or near real
time, often from different locations. Choose the tool category that matches the
work:

- **Spreadsheets** organize calculations and structured data in rows and
  columns, with shared editing when supported.
- **Videoconferencing** supports live meetings with remote participants.
- **Presentation tools** create and deliver slide-based visual material.
- **Word processing tools** create and revise text-focused documents.
- **Instant messaging** supports quick text conversations and short updates.

Several functions may appear in one suite, but the scenario's immediate task
still determines the best tool category. (CompTIA Objective 1.11; Professor
Messer, p. 23)

## Identity Synchronization

Organizations may have a local directory and one or more cloud identity
providers. Identity synchronization propagates directory changes so an account
created or updated in the managed source can appear in connected directories.
This reduces repeated account maintenance and gives administrators a consistent
place to manage common identity information. (Professor Messer, p. 23)

Synchronization does not by itself grant every cloud application. The user may
have a synchronized identity but still need the appropriate license assigned.

## Licensing Assignment

Cloud productivity licenses are commonly assigned to users rather than tied to
one physical computer. Centralized license management lets an administrator see
assignments and move an available license from one user to another. When a
newly synchronized user can sign in but cannot use a licensed application,
verify the user's license assignment. (Professor Messer, p. 23)

## Configuration Decision Sequence

1. Identify whether the user needs hosted email, storage, or a collaboration
   function.
2. Configure the required storage folders and choose local or streaming access
   according to device capacity and offline needs.
3. Confirm that identity synchronization has created or updated the expected
   cloud identity.
4. Assign the required cloud service or application license to that user.
5. Verify that the user can access the intended service from the appropriate
   device.
